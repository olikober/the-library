## Execution Instructions

### 1. Read and Understand

- Read the ENTIRE plan carefully
- Understand all tasks and their dependencies
- Note the validation commands to run
- Review the testing strategy

### 2. Analyze Task Dependencies

Before creating the todo list, analyze dependencies between tasks:
- Identify which tasks can run independently (no dependencies on other tasks)
- Identify which tasks depend on others (must run sequentially)
- Group independent tasks that can execute in parallel

### 3. Create Todo List

**Initialize progress tracking** by creating a todo list from the plan tasks:

Use `manage_todo_list` with `operation: "write"` to track all tasks from the plan:
- Convert each "Step by Step Task" into a todo item
- Set initial status: `not-started` (or `in-progress` for first task)
- Include task description and relevant file paths in each item
- Add a `parallel_group` field to identify which tasks can run together

Example:
```json
[
  {
    "id": 1,
    "title": "Create user authentication module",
    "description": "Implement login/logout with JWT tokens in src/auth/",
    "status": "in-progress",
    "parallel_group": 1
  },
  {
    "id": 2,
    "title": "Add database schema for users",
    "description": "Create users table in src/db/schema.ts",
    "status": "not-started",
    "parallel_group": 1
  },
  {
    "id": 3,
    "title": "Create API routes for users",
    "description": "Implement REST endpoints in src/routes/users.ts",
    "status": "not-started",
    "parallel_group": 2
  }
]
```

### 4. Execute Tasks with Parallel Optimization

#### a. Identify Parallel Execution Opportunities

Before executing, determine which tasks are truly independent:
- Tasks with no file-level dependencies on each other
- Tasks that don't modify shared state or configuration
- Tasks that can run concurrently without conflicts

**Examples of independent (parallelizable) tasks:**
- Creating separate component files
- Setting up different utility modules
- Creating independent API route handlers
- Adding separate test files

**Examples of dependent (sequential) tasks:**
- Task B reads/output from Task A
- Task B modifies files that Task A also modifies
- Task B requires artifacts generated by Task A

#### b. Execute Independent Tasks in Parallel

For tasks identified as independent, use the subagent tool with `tasks: [...]` (PARALLEL mode):

```json
{
  "tasks": [
    {
      "agent": "default",
      "task": "Implement the user authentication module. Create src/auth/index.ts with login/logout functions using JWT tokens."
    },
    {
      "agent": "default", 
      "task": "Create the database schema. Add users table to src/db/schema.ts with fields: id, email, password_hash, created_at."
    }
  ]
}
```

**Parallel execution guidelines:**
- Group 3-5 independent tasks per parallel batch
- Pass clear, self-contained task descriptions to each subagent
- Include relevant file paths and requirements in each task
- Use `cwd` parameter if tasks need specific working directory

#### c. Execute Dependent Tasks Sequentially

For tasks with dependencies, execute sequentially:
- Complete the prerequisite task first
- Verify completion before starting dependent task
- Update todo list after each sequential task

#### d. Update Progress

- Mark completed tasks using `manage_todo_list`
- For parallel tasks: mark all as completed after subagent returns
- Keep todo list synchronized throughout execution

#### e. Verify as you go

- After each batch (parallel or sequential), verify changes
- Check syntax and imports
- Ensure types are properly defined

### 5. Implement Testing Strategy

After completing implementation tasks:

- Create all test files specified in the plan
- Implement all test cases mentioned
- Follow the testing approach outlined
- Ensure tests cover edge cases

**Parallel testing:** If multiple test files are independent, consider parallel execution:
```json
{
  "tasks": [

    { "agent": "default", "task": "Create unit tests for db schema in tests/db.test.ts" }
  ]
}
```
